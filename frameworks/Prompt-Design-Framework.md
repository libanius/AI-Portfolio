# Prompt Design Framework

This framework describes how I design prompts and AI agents for practical workflows.

## 1. Define the Goal

What should the AI accomplish?

Examples:

- Qualify a lead
- Answer customer questions
- Evaluate a response
- Generate a project update
- Guide a user through a decision

## 2. Define the User Context

Who is the user?

- Customer
- Contractor
- Business owner
- Student
- Homeowner
- Project manager

## 3. Define Constraints

What should the AI avoid?

Examples:

- Do not invent prices
- Do not give legal advice as certainty
- Do not overpromise
- Escalate complex cases
- Ask clarifying questions only when needed

## 4. Define Decision Logic

What should happen depending on the user's answer?

Example:

- If the customer is ready to schedule, collect availability.
- If the customer asks about price, provide range and explain variables.
- If the issue is complex, escalate to a human.

## 5. Define Tone

The AI should match the business and user.

Examples:

- Professional
- Clear
- Friendly
- Direct
- Bilingual when needed

## 6. Define Success Criteria

A good AI response should be:

- Helpful
- Accurate
- Clear
- Actionable
- Safe
- Consistent with business rules

## 7. Test and Improve

I improve prompts by checking:

- Did the AI follow instructions?
- Did it ask unnecessary questions?
- Did it miss important context?
- Did it hallucinate?
- Did it sound natural?
- Did it help the user move forward?
